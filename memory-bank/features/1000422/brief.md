# Проблема

Ограничения новой роли Аудитор заданы в ролевой модели, но не обеспечены при выдаче данных в двух Elixir-сервисах: пользователь с этой ролью теоретически может получить объём данных шире, чем предусмотрено политикой доступа. **Сейчас ограничения Аудитора не подключены к данным в двух сервисах на Elixir**; (обзор и часть «конвертер» — в [spec.md](spec.md)).

**Критерий снятия проблемы:** для каждого сервиса из согласованного перечня подтверждено, что выдача данных для роли Аудитор соответствует ограничениям ролевой модели (проверка по чек-листу сервисов и сценариев).

# Контекст

На проекте действует ролевая модель с правами и ограничениями по ролям. Вводится роль Аудитор с ограничениями на просмотр данных в контурах: журналы, отчёты. Команда отвечает за все сервисы на Elixir, через которые эти данные отдаются клиентам.

Релиз функциональности Аудита запланирован на 25.04.2026. К этой дате ограничения роли Аудитор должны соблюдаться при выдаче данных во всех **целевых** Elixir-сервисах.  **Структура технической спеки:** [spec.md](spec.md).

# Требования (желаемый результат)

Во всех целевых сервисах на Elixir пользователь получает только те данные, которые соответствуют ограничениям своей роли, в том числе для роли Аудитор. Допустимые способы реализации задаются спекой (Ecto, библиотека zeus, привязка полей PDP к схемам в сервисах); детали по слоям — в [spec.md](spec.md).
